2. 1주차 주요 개념 키워드 적어보기
1) sourcetree
 -> 와 진짜 편하다. 한눈에 보고 따라하기 너무 편함
2) init, add, commit, push, pull
 2-1) init
  -> 프로젝트를 만들고 git 프로젝트로 만들기 위한 첫 작업
  -> sourcetree -> create -> 내가 만든 프로젝트 폴더 경로 설정
 2-2) add
  -> 내가 변경한 부분들을 commit하기 위해 실행하기 전 단계, stage에 올려둔다고 해서 staging이라고도 표현
  -> 변경된 파일에서 + 버튼 클릭
 2-3) commit
  -> add 된 파일을 저장하는 단계
 2-4) push
  -> 로컬 repo( 내 컴퓨터에서 작업한 git 프로젝트 )에서 원격 repo( GitHub ) 로 올리는 작업
 2-5) pull
  -> 원격 repo( GitHub )에서 로컬 repo( 내 컴퓨터에서 작업한 git 프로젝트 ) 로 올리는 작업

3. Git VS SVN
 -> 매우 충격적이다. SVN으로 작업할 때는 너무나 힘들었는데, 왜 Git이 개발자에게 더 최적화가 되었다는 것을 느낄 수 있었음
 -> 직관적인 명령어 ( push, pull ) 등으로 이해하기가 너무 쉬움, 소스 관리가 정말 편할 듯 ( SVN으로 작업할 때는... )
